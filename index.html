Protocol stack

Four composable modules, one protocol.

Presence Protocol isn’t a single mining app—it’s an infrastructure stack with explicit boundaries: encounter capture, relay, verification, and SDK tooling. Mix-and-match modules or build the next layer on top.

Presence Engine

Deterministic BLE handshake converting mutual proximity into signed encounters.

Relay Network

Store/forward infrastructure for offline-first sync, fraud checks, and witness attestations.

Reward Verification

Backends enforce invariants, execute cooldown policy, and issue rewards only after verification.

Builder SDK

Typed APIs + tooling for mining apps, diagnostics dashboards, and third-party integrations.

Telemetry flow

Deterministic encounter lifecycle.

Scan

Duty-cycled BLE scanning discovers other Presence devices.

Handshake

Mutual key exchange + signatures prove proximity without GPS.

Queue

Immutable payload stored locally until connectivity returns.

Verify

Backend enforces signatures, cooldown policies, and telemetry rules.

Reward

Accepted encounters issue receipts; telemetry emits state changes.

Builder ecosystem

SDKs, telemetry contracts, and reference apps.

Reference apps (Presence Miner, Relay, Wallet) prove the protocol. Builders can fork them or start from low-level SDKs. Every module ships with deterministic tests, KCS-aware fee logic, and telemetry events.
